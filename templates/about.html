About this project

This project is a Rust-powered evolution of Andrej Karpathy's original arxiv-sanity-preserver, focused on helping you track and prioritize research papers.

What this project is

  • A paper discovery interface for recent arXiv categories (for example: cs.CV, cs.CL, cs.LG, cs.AI, cs.NE, stat.ML).
  • A personal research workflow tool with search, filtering, recommendations, and a saved library.
  • A local-first app that can run on your own machine or server.

What was modified vs. Karpathy's original version

  • Backend rewritten in Rust (Axum), replacing the original Flask/Python server flow.
  • Modernized UI and layout with updated toolbar, filtering panels, theme toggle, and settings modal.
  • Pipeline integration for ingest, recompute status, and operational controls in the interface.
  • Improved topic-based download settings and richer metadata display in settings.

How the project works (high level)

  1. Ingest: paper metadata is downloaded and normalized into local storage/database.
  2. Index: text features are generated and a nearest-neighbor index is built.
  3. Serve: the web server loads cache/index/database and renders the browsing UI.
  4. Interact: users search, filter by topics, save papers, and view recommendations.

APIs and endpoints

Examples of app endpoints include:

  • / — home/recent papers
  • /library — saved papers
  • /search — query and filter results
  • /topics, /top, /recommend — discovery/recommendation views
  • /ingest, /ingest/status/:job_id, /recompute/status — pipeline operations
  • /settings/download and /settings/download/run — topic download settings and execution

Database and similarity

  • The app persists core user and library data in a local SQLite database (.pipeline/as.db).
  • Search and recommendation data are loaded from local cached artifacts and indexes.
  • Similarity/recommendation relies on vector/text features and nearest-neighbor retrieval for related papers.
